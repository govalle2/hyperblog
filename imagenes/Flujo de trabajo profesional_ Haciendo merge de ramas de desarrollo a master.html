      Hola, me paso lo mismo para esto encontre documentación que me ayudo a solucionarlo.
      Notarás la frase “Fast forward” (“Avance rápido”, en inglés) que aparece en la salida del comando. Git ha movido el apuntador hacia adelante, ya que la confirmación apuntada en la rama donde has fusionado estaba directamente arriba respecto a la confirmación actual. Dicho de otro modo: cuando intentas fusionar una confirmación con otra confirmación accesible siguiendo directamente el historial de la primera; Git simplifica las cosas avanzando el puntero, ya que no hay ningún otro trabajo divergente a fusionar. Esto es lo que se denomina “avance rápido” (“fast forward”).
      En el caso de que requiera una confirmación de fusión durante una combinación de avance rápido ejecutar git merge con --no-fast forward
      Entonces puedes darle un git reset --hard <commit anterior>
      Así retrocedes y vuelves a ejecutar el merge.

      git merge --no-ff <branch>

      Este comando combina la rama especificada en la rama actual, pero siempre genera una confirmación de fusión (incluso si se trata de una fusión de avance rápido). Esto es útil para documentar todas las fusiones que se producen en su repositorio.

      Flujo de trabajo profesional: Haciendo merge de ramas de desarrollo a master
      Si queremos usar imágenes o cualquier archivo binario al blog no debemos de incluirlas en el repositorio, deben ser ignoradas.
      Si se agrega alguno de estos archivos al repositorio y luego hacemos push al repositorio remoto veremos que podemos ver la imagen en GitHub, el problema es que pesa mucho. Si es un solo archivo no pasa nada, pero cuando se tienen más es un problema tener que moverlos cada que se hace un pull o push al repositorio.

      Si editamos la imagen y vemos el status, Git va a detectar que la imagen fue modificada, pero no podrá decirnos exactamente que fue lo que se modifico porque es un archivo binario.

      En la rama Master solo se debe enviar lo que está listo para producción.

      Cuando múltiples desarrolladores están trabajando pueden hacer pull a otras ramas y ver lo que han hecho los demás.
